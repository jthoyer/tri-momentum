Components
Buttons & chips
Primary actions use the solid CTA; everything else is a chip, tag, or ghost button so the hierarchy stays obvious at a glance. All buttons share one 10px corner radius (--rmd) — no more pill-shaped chips at 20px. Primary buttons are never shown disabled: gate progress with inline validation instead of a CTA the user can't tell is broken.

Guidelines
Accessibility notes
Constraints baked into the tokens above — worth restating so future components don't regress them.
Text size
Base body is 16px. Nothing below 12px is used for reading content — the only sub-12px elements are single-glyph decorative icons (chevrons, delete ×).
Touch targets
Interactive controls hold a 44×44px minimum (WCAG 2.5.5). Where a visual element is smaller, it shares its parent's hit area rather than shrinking the target.
Contrast
--ink2 and --ink3 on white or --s3 both clear 4.5:1 (WCAG AA) for normal text.
Zoom
The viewport meta tag sets initial-scale=1.0 only — no maximum-scale or user-scalable=no, so pinch-zoom always works.
No disabled CTAs
A primary button that looks clickable but silently does nothing is a dark pattern. Missing input should surface as a visible message, not a disabled affordance.
